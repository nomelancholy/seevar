# 경험치(XP) 및 레벨 시스템

See VAR의 유저 경험치·레벨 규칙을 정의한 문서입니다. 구현: `lib/utils/xp.ts`, `lib/actions/moments.ts`, `lib/actions/referee-reviews.ts`.

---

## 1. 경험치 획득 로직 (XP 구조)

가중치를 두기 위해 **모멘트 생성**을 '주요 활동', **심판 평가**를 '일상 활동'으로 구분합니다.

| 활동 유형 | 액션 | 획득 경험치 | 가중치 이유 |
|-----------|------|-------------|-------------|
| **모멘트 생성** | 경기 장면 공유 및 글 작성 | **50 XP** | 콘텐츠 생산 및 커뮤니티 활성화의 핵심 |
| **심판 평가** | 경기 후 심판 평점 부여 | **10 XP** | 간단한 참여형 액션 |
| **연속성 보너스** | 3경기 연속 평가 참여 | **+20 XP** | 유저 리텐션(유지율) 강화 |
| **베스트 모멘트** | 내가 만든 모멘트가 추천(SEE VAR)받음 | **+10 XP** | 양질의 콘텐츠 생산 유도 |

- **연속성 보너스**: 같은 라운드 내에서 서로 다른 경기 3개에 심판 평가를 남기면, 3번째 평가 저장 시 한 번 +20 XP가 지급됩니다.
- **베스트 모멘트**: 다른 유저가 내 모멘트에 SEE VAR를 누를 때마다 작성자에게 +10 XP가 지급됩니다. (본인이 누른 SEE VAR는 제외)
- **심판 평가 수정**: 이미 남긴 평가를 수정(재저장)할 때는 XP가 추가 지급되지 않습니다. 최초 작성 시에만 10 XP 지급.

## 2. 레벨 테이블 설계 (Level Curve)

레벨이 올라갈수록 **다음 레벨까지 필요한 경험치가 늘어나는 지수 성장형** 모델을 사용합니다.  
초반에는 성취감을 위해 비교적 빠르게 레벨업되도록 설계했습니다.

**공식**: 다음 레벨 필요 XP = **현재 레벨 × 100**

| 레벨 구간 | 누적 XP 범위 | 다음 레벨까지 필요 XP | 비고 |
|-----------|--------------|------------------------|------|
| Lv.1 (뉴비) | 0 ~ 100 | 100 | 모멘트 2번이면 레벨업 |
| Lv.2 (팬) | 100 ~ 300 | 200 | |
| Lv.3 (서포터) | 300 ~ 600 | 300 | |
| … | … | … | |
| Lv.10 (전문가) | 4,500 ~ 5,500 | 1,000 | |
| Lv.99 (판정관) | — | — | 만렙 |

- **누적 XP**로 레벨이 결정됩니다.
- Lv.1은 0 XP부터, Lv.2는 100 XP 이상, Lv.3은 300 XP 이상, … 순으로 구간이 이어집니다.

---

## 3. 기타 규칙

- **신고 승인(숨김)**: 댓글/평가가 신고 승인으로 숨김 처리될 때, 작성자 XP가 일정 수치 차감됩니다. 재심 통과로 다시 노출되면 같은 수치만큼 복구됩니다. (상수: `XP_PENALTY_ON_HIDE`)
- **심판 평가 답글**: 평가에 답글을 달면 소량 XP가 지급됩니다. (상수: `XP_REVIEW_REPLY_CREATE`)
